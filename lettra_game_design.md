# LETTRA the Postman — ゲーム設計仕様

決定事項を随時追記する作業ドキュメント。未決事項は末尾の「Open Questions」に。

## コアコンセプト

- ビブリボン的横スクロール音ゲー × MPC的チョップ&フリップ
- 主人公=配達員、走行は自動、プレイヤー入力はサンプル発音のみ
- 「叩くと曲が完成する」体験。死なない設計(叩かないと曲がスカスカになるだけ)

## グラフィック方針

- ビブリボン継承のローポリ幾何路線
- 線画 + 単色フィル(活版印刷=Crane's Lettra の手触り)
- パレット: 紙白 / インク黒 / ポスト赤 / ステージ別1色アクセント
- 地形は単純ポリゴン(三角・矩形・円)の連続
- TT モノグラム(LETTRA ロゴの中央TT)が地形オブジェクトとして登場 = ブランドとゲームの結節点

## 音楽システム

### 構造

```
[オケトラック / 常時再生・固定]
  └ ドラム
  └ ベース
[プレイヤー演奏領域 / ボタン入力で発音]
  └ コード stab
  └ メロディ片
  └ ボーカルチョップ
  └ SE系サンプル
```

ドラムはBGMではなくバッキング。プレイヤーは MPC のパッドを叩く側。

### ボタン: 4(2x2 MPC配置)

物理ボタンは右側面の○×△□ / ABXY を 2x2 として使用。十字キーは未使用(走行は自動)。

音域/役割を物理位置にマッピング:

```
   △ 上=高音(メロディ・ベル)
 □           ○
 左=ハーモニー  右=アクセント
 (コードstab) (ボーカルチョップ・SE)
   × 下=低音(ベース被せ・キックレイヤー)
```

ブロック切替でサンプルが入れ替わっても、この役割マッピングは維持する。プレイヤーが「上は高い音」と直感で叩けることが重要。

### サンプルアサイン: 4or8小節単位で切替

- 1ステージ = 90秒 ≒ 33小節(BPM 90想定)
- 8小節区切りで **4ブロック** が基本単位
- 1ステージあたり **4ブロック × 4ボタン = 16サンプル**(実質はフリップ・使い回しで8〜10)
- 1ステージ = 1ビートキットを組む作業 ≒ MPCで1曲組むのと同等

#### ブロック構成の標準形

| Block | 範囲 | 役割 | 内容例 |
|---|---|---|---|
| 1 | 0-8小節 | intro | 静かなパッド系 |
| 2 | 8-16小節 | verse | コードstab + ボーカルチョップ |
| 3 | 16-24小節 | chorus | メロディ片 + ブラス |
| 4 | 24-32小節 | outro | Block1の逆再生フリップ |

#### ブロック切替の演出(地形と完全同期)

- 切替の1〜2小節前に**ポストの色が変わり始める**(予告)
- 切替フレームで**画面アクセントカラーが変わる**
- 配達員が**ゲート(郵便局/分岐点/橋)**を通過 = ブロック境界
- ゲートをくぐると4ボタンのサンプルが入れ替わる

### 譜面密度

| Block | 役割 | 打数(目安) | 推奨クオンタイズ |
|---|---|---|---|
| 1 | intro | 8 | どちらでも |
| 2 | verse | 20 | ON |
| 3 | chorus | 35 | OFF |
| 4 | outro | ~15(減衰) | OFF |

合計 約90打 / 90秒 = 1秒1打弱。叩きすぎないのが MPC 的。

### クオンタイズ ON/OFF

- プレイヤーが**プレイ中に切替可能**(L1/L2 トグル想定)
- 切替時に画面端に小さく `Q ON` / `Q OFF` を表示
- スコア計算は別系統:
  - ON: ジャスト判定が緩い、安全
  - OFF: ジャスト幅が狭いがボーナス
- リプレイ時に**ブロック単位でON/OFF履歴が記録**される(プレイスタイルが残る)

## ステージ仕様

- 1ステージ = 1曲 = 1通の配達物語
- 長さ: **90秒**
- 構成: 4ブロック(8小節 × 4)

## 失敗ハンドリング

- 死なない。ステージは必ず最後まで走り切れる
- ミス = 手紙が風に飛ぶ / ポストを通り過ぎる
- サンプルが鳴らない = オケのドラム+ベースだけが続く
- **連続ミスでもオケは何も変わらない**(ベースもドラムも抜けない)
  - プレイヤーの貢献感はサンプル発音の有無だけで表現
  - 「曲が崩壊する恐怖」ではなく「叩くと曲が彩られる喜び」に倒す

## ブロック境界の扱い

- ブロック切替直前の入力は**前ブロックのサンプル**で発音される
- 切替の違和感は**ドラムフィル**等で吸収(オケ側で境界をデザイン)
- 譜面上もブロック境界1〜2拍前に密度を落とすか、フィルに同期した配置にする

## リリース戦略

- **無料公開フェーズ**: 1ステージずつ増やしながら配布
- **Steam公開フェーズ**: 10曲超え + ファンがついた段階で有料パッケージ化
- ワールド/アルバム概念は Steam 版で導入(無料期間は1曲単位の積み上げ)

## 判定仕様

### クオンタイズ OFF 時

- 許容幅: **±60ms**
- 範囲内: 押した瞬間のタイミングでサンプル発音(ズレたまま鳴る = MPC的気持ちよさ)
- 範囲外: **不発**(ダサく鳴らさない、無音で「外した」を伝える)

### クオンタイズ ON 時

- 許容幅: 別途定義(OFFより緩く)
- 範囲内: 直近のグリッドにスナップして発音
- 範囲外: 不発
